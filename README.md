# Application of Deep Reinforcement Learning for Aircraft Control using Behavior Trees

This is a DRDO-inspired academic project focused on building a simplified 2D aircraft simulation in Python. The control architecture combines **Deep Reinforcement Learning (DRL)** for low-level flight control and maneuvers (e.g., evasion, recovery, waypoint tracking) with a **Behavior Tree (BT)** for high-level decision-making and switching states.

---

## Project Directory Structure

Here is an overview of the directory structure we have created and the purpose of each directory:

*   **`src/`**: This is the main directory containing all the Python source code for the project. Keeping our code inside a dedicated source folder ensures clean project organization.
    *   **`src/simulation/`**: Contains the code responsible for the 2D physics simulation, drawing graphics, and handling objects.
        *   *Aircraft Class*: Manages aircraft movement, acceleration, turning, and kinematics.
        *   *Missile Class*: Homing logic for the missile threat.
        *   *Simulation Loop*: Handles the Pygame window, coordinates, update steps, and drawing.
    *   **`src/rl/`**: Contains all Reinforcement Learning logic.
        *   *Gymnasium Environment*: Wraps our simulation into a standard Gym interface with step, reset, observation, and action methods.
        *   *Reward Functions*: Calculates the numerical penalty or reward for the aircraft's performance (e.g., staying within boundaries, avoiding missiles, reaching waypoints).
    *   **`src/behavior_trees/`**: Contains behavior tree configurations and code.
        *   *Tree Logic*: Orchestrates high-level states (e.g., Patrol, Navigate, Evade, Recover) and switches control between them.
*   **`models/`**: A folder to save trained neural network models (e.g., `.zip` files containing the policy weights) generated by Stable-Baselines3.
*   **`notebooks/`**: A workspace folder for Python notebooks or script files to analyze, plot, and verify trajectories, state changes, and reward trends.
*   **`requirements.txt`**: Lists all external Python libraries required to install and run this project.

---

## Explanation of Libraries Used

Here is why each package listed in `requirements.txt` is required for this project:

1.  **`pygame`**:
    *   *What it is*: A standard cross-platform Python library designed for writing video games and multimedia applications.
    *   *Why we use it*: It provides the 2D window rendering, basic physics ticks, key inputs, and drawing functions (e.g., drawing the aircraft and missile as 2D vectors/shapes) so we can visually see what the aircraft is doing.
2.  **`gymnasium`**:
    *   *What it is*: An active fork of OpenAI's Gym library. It is the standard interface in the RL community for defining custom environments.
    *   *Why we use it*: Reinforcement Learning agents learn by interacting with environments. Gymnasium provides a standardized template containing predefined functions (`reset`, `step`, `render`) that allow Stable-Baselines3 agents to seamlessly connect to our Pygame simulation.
3.  **`stable-baselines3`**:
    *   *What it is*: A set of reliable, highly optimized, and tested implementations of Reinforcement Learning algorithms in PyTorch.
    *   *Why we use it*: Implementing algorithms like PPO (Proximal Policy Optimization) from scratch is error-prone and complex. We use this library to leverage a production-grade, bug-free implementation of PPO.
4.  **`torch` (PyTorch)**:
    *   *What it is*: An open-source machine learning library primarily developed by Meta's AI Research lab.
    *   *Why we use it*: PyTorch acts as the backend for Stable-Baselines3. It represents the neural networks (policies) that compute flight control commands and optimizes them during training.
5.  **`behavior3py`**:
    *   *What it is*: A Python port of the Behavior3 library, which is a formal framework for designing and running Behavior Trees (BTs).
    *   *Why we use it*: Behavior Trees provide structured, deterministic, and readable high-level task switching (e.g., "If missile is detected, switch active controller to Evade"). It acts as the "brain's supervisor."
6.  **`numpy`**:
    *   *What it is*: The fundamental library for scientific and mathematical computing in Python.
    *   *Why we use it*: It provides multi-dimensional arrays and fast mathematical functions, allowing us to compute distances, angles, coordinate projections, and vector rotations quickly.
7.  **`matplotlib`**:
    *   *What it is*: A comprehensive library for creating static, animated, and interactive visualizations in Python.
    *   *Why we use it*: After training or during simulation runs, we will use it to plot graphs showing training progress (reward vs. episode) and trajectory paths to analyze how the aircraft avoided the missile.
